# APC 통신 체커 (APC Device Communication Checker)

## 📌 개요
다양한 통신 장비(Slave)와의 통신 호환성 및 프로토콜을 테스트하기 위한 전용 애플리케이션입니다. 
장비마다 다른 통신 규격과 데이터를 손쉽게 관리하고 검증할 수 있도록 지원합니다.

## ⚙️ 주요 기능 및 특징

### 1. 통신 방식 및 프로토콜 선택
- **통신(인터페이스) 선택**: 사용자가 현재 테스트하고자 하는 통신 방식을 선택합니다. (예: DeviceNet 등)
- **프로토콜(Config) 선택**: 선택한 통신 환경 내에서 어떤 프로토콜 설정을 테스트할지 지정할 수 있습니다.

### 2. JSON 기반 프로토콜 설정 관리
- 모든 통신 프로토콜은 `config` 폴더 내에 직관적인 **JSON 파일** 형태로 정의됩니다.
- 통신 패킷 스펙(자료형, 이름, 비트맵, Enum 등)을 JSON 파일에 구조화하여, 코드 수정 없이 프로토콜을 확장시킬 수 있습니다.

### 3. 사용자 설정(유저 디파인) 지원
- **사용자 정의 프로토콜 추가**: 사용자는 장비에 맞는 새로운 통신 프로토콜(JSON 파일)을 신규로 정의할 수 있습니다.
- **프로토콜 수정**: 각 프로토콜의 구성 내용(세부 스펙 등)을 앱 내에서 직접 수정할 수 있습니다.
- **다른 이름으로 저장**: 기존에 만들어둔 프로토콜 설정 문서를 복사(다른 이름으로 저장)하여, 비슷한 장비의 테스트 설정을 빠르게 구축할 수 있습니다.

## 📁 구조 요약
- `config/`: JSON 기반의 프로토콜 설정 파일이 저장되는 공간
- `model/`: 통신 데이터를 파싱하고 저장하기 위한 내부 데이터 모델(Pydantic 기반)
- `dnet_checker.py`: 메인 실행 파일 및 초기 테스트 스크립트
